## 1. Project Goal

PhishGuard is a **Gmail-based email security pipeline**.
The long-term goal is to automatically detect phishing emails (internship/job scams) and surface them to the user.

| Phase | Status | Description |
|---|---|---|
| **Phase 1** | ✅ COMPLETE & VERIFIED | End-to-end ingestion plumbing — Pub/Sub webhook → Gmail fetch → label apply |
| **Phase 2** | 🔵 DESIGNED, NOT YET BUILT | L1/L2/L3 security analysis pipeline with SCL scoring |
| **Phase 2.5** | 📋 PLANNED | Warning draft injection into Gmail thread on PHISH verdict |

## 5. Phase 1 — File-by-File Breakdown (existing, do not break)

### `config.py`
- Loads `.env` via `python-dotenv` at import time
- **All other modules import from here** — never read `os.getenv` directly elsewhere
- Phase 1 exports: `GOOGLE_CLIENT_CREDENTIALS_PATH`, `GOOGLE_TOKEN_PATH`, `PUBSUB_TOPIC_NAME`, `GMAIL_USER_ID`, `TEST_TRIGGER_WORD`

### `gmail_client.py`
- Manages OAuth 2.0 using `google-auth-oauthlib` + `google-api-python-client`
- Scope: `https://www.googleapis.com/auth/gmail.modify`
- **First run:** opens browser for consent → writes `token.json`
- **Subsequent runs:** loads `token.json`, auto-refreshes if expired
- Public API: `get_gmail_service()` → returns an authenticated Gmail API `Resource`
- If `credentials.json` is missing, raises `FileNotFoundError` with an actionable message

### `label_manager.py`
- Manages the `[PhishGuard]` Gmail label (Phase 1)
- `ensure_label_exists(service)` — called at startup; creates label if absent
- `apply_phishguard_label(service, message_id)` — adds the label via `users.messages.modify`
- Label ID is **cached in memory** (`_label_id_cache`) — only one API call per process lifetime
- Returns `bool` — `True` on success, `False` on `HttpError` (logs the error, never raises)
- **Phase 2 extension:** Add `[PhishGuard] Suspicious` and `[PhishGuard] Phishing` labels here

### `watch_manager.py`
- `register_watch(service)` — calls `users.watch` with `labelIds: ["INBOX"]`
- Requires `PUBSUB_TOPIC_NAME` to be set — raises `ValueError` if missing
- Logs `historyId` and `expiration` from the response
- Returns the full watch response dict

### `main.py`
- FastAPI app using the modern `lifespan` context manager
- **Startup sequence:** authenticate → ensure labels → register watch
- **Module-level state:** `gmail_service`, `last_history_id` (cursor for `history.list`)
- **Webhook:** `POST /webhook/gmail` — decodes Pub/Sub envelope, calls `history.list`, processes each new message
- **Always returns HTTP 200** — prevents Pub/Sub redelivery storms
- **Phase 2 change:** `_process_message` is replaced with async pipeline call

## 6. Phase 2 — Security Pipeline Design

### The SCL Scoring Model (mirrors Microsoft EOP)

| SCL Score | Verdict | Action |
|---|---|---|
| 0–3 | `CLEAN` | No action |
| 4–6 | `SUSPICIOUS` | Apply `[PhishGuard] Suspicious` label |
| 7–8 | `LIKELY_PHISH` | Apply `[PhishGuard] Phishing` label + log warning |
| 9–10 | `PHISH` | Apply `[PhishGuard] Phishing` label + move to SPAM folder |

> **Note on subject modification:** Gmail messages are immutable — the subject cannot be edited via the API. For LIKELY_PHISH, the label is applied and a log warning is emitted. Phase 2.5 (warning draft) supersedes this entirely.

---

### Stage 1 — `pipeline/email_parser.py`

**Output:** `EmailContext` dataclass

```python
@dataclass
class EmailContext:
    message_id: str
    thread_id: str          # needed for Phase 2.5 draft threading
    subject: str
    sender: str             # full From header value
    sender_domain: str      # extracted domain from From address
    reply_to: str
    to: str
    date: str
    body_text: str          # plain text, HTML stripped via BeautifulSoup
    body_html: str          # raw HTML if present
    headers: dict           # full dict of all raw headers
    attachments: list[dict] # [{filename, mime_type, content_bytes}]
    urls: list[str]         # deduplicated URLs from body + href attrs
    raw_mime: str
```

- Uses Python's built-in `email` library for MIME parsing
- `BeautifulSoup` strips HTML to plain text
- URLs extracted via both `re` (bare URLs) and `html.parser` (href attributes)

---

### Stage 2 — `pipeline/l1_authentication.py`

Mirrors EOP L1 edge filter. **Max contribution: 4 points.**

| Check | Method | Score |
|---|---|---|
| SPF | `dnspython` TXT lookup on sender domain | fail→+3, softfail→+1, none→+1 |
| DKIM | Parse `Authentication-Results` header | fail/missing→+2 |
| DMARC | Parse `Authentication-Results` header | fail→+3, none→+1 |
| Display name spoofing | Regex keywords (admin, security, helpdesk, IT support, university, professor) vs actual domain | match→+2 |

**Signature:** `check_authentication(ctx: EmailContext) -> tuple[int, list[str]]`

> **SPF note:** Gmail rewrites `Received:` headers for forwarded mail; result will often be `none` — this is expected and intentional (fail-open).

---

### Stage 3 — `pipeline/l2_heuristics.py`

Deterministic rule engine. **Max contribution: 4 points.**

| Rule | Trigger | Score |
|---|---|---|
| `reply_to_mismatch` | Reply-To domain ≠ From domain | +2 |
| `pivot_to_external` | WhatsApp/Telegram/"contact me at" + external handle | +2 |
| `urgency_language` | "act now", "account suspended", "verify immediately" | +1 |
| `credential_harvesting` | "click here to verify", "confirm your password", "login to restore" | +2 |
| `blank_body_with_attachment` | Body < 30 chars AND attachment present | +3 |
| `suspicious_sender_pattern` | `no-reply-`/`security-` prefix + free email domain | +1 |
| `free_domain_impersonation` | Free provider domain + institutional display name | +2 |
| `excessive_urls` | > 5 URLs in body | +1 |
| `newly_registered_domain_pattern` | Shannon entropy of domain name > 3.5 | +1 |

**Signature:** `check_heuristics(ctx: EmailContext) -> tuple[int, list[str]]`

---

### Stage 4 — `pipeline/l2_nlp.py`

TF-IDF + Logistic Regression classifier. **Max contribution: 3 points.**

- Model loaded at import time from `models/vectorizer.pkl` + `models/classifier.pkl`
- Fails gracefully if models not found (logs warning, returns score 0) — never crashes the pipeline
- `classify_email(body_text: str) -> tuple[str, float]` returns `("phish"|"ham", confidence)`
- Score: confidence > 0.85 → +3 | 0.60–0.85 → +2 | < 0.60 → +1

**Training script:** `models/train_classifier.py` (one-time, not imported at runtime)
- Dataset: CSV with `text` and `label` columns in `data/` (CEAS 2008 or Nazario corpus)
- Preprocessing: lowercase → strip HTML → replace URLs with `URL_TOKEN` → strip emails
- `TfidfVectorizer(ngram_range=(1,2), max_features=10000)` + `LogisticRegression(max_iter=1000)`
- Saves with `joblib`; prints classification report + confusion matrix

---

### Stage 5 — `pipeline/mcp_client.py` + `pipeline/l3_threat_intel.py`

Mirrors MDO Safe Links / VirusTotal integration. **Max contribution: 3 points** (capped).

#### `mcp_client.py` — MCP stdio client

- Spawns `VT_MCP_SERVER_COMMAND` + `VT_MCP_SERVER_ARGS` as a **persistent child process** using `asyncio.create_subprocess_exec`
- Performs the `initialize → initialized` MCP handshake on first use
- Communication: JSON-RPC over stdin/stdout (MCP stdio transport protocol)
- **One subprocess per app lifetime** — do not spawn per email
- Auto-incrementing request ID to correlate responses
- Public API: `async call_mcp_tool(tool_name: str, arguments: dict) -> dict`
- Timeout: 10 seconds per call — returns `{"error": "timeout"}` rather than raising

#### `l3_threat_intel.py` — Three async VT checks

| Function | MCP Tool Called | What It Checks | Score Logic |
|---|---|---|---|
| `scan_url(url)` | `scan_url` | 70+ engine consensus verdict | malicious≥5→+3, 1–4→+2, suspicious≥3→+1 |
| `scan_domain(domain)` | `get_domain_report` | Domain reputation, votes, creation date | reputation<−20→+2, votes≥3→+2, seen<30d→+1 |
| `scan_file_hash(filename, mime, bytes)` | `get_file_report` | SHA-256 hash lookup (no upload) | malicious≥3→+3, 1–2→+2, suspicious≥2→+1 |

**Attachment filter:** `scan_file_hash` only called for: `application/pdf`, `application/zip`, `application/vnd.openxmlformats*`, `application/octet-stream`

**In-memory cache:**
```python
_vt_cache: dict[str, dict] = {}  # key: url/hash/domain → cached result
```

**Token bucket rate limiter:** Tracks calls in last 60 seconds. Free tier = 4/min. Awaits `asyncio.sleep()` if at limit.

**Structured log format:**
```
[L3-VT] [MSG_ID] url=https://... engines=72 malicious=7 score=+3
[L3-VT] [MSG_ID] domain=suspiciousdomain.xyz reputation=-45 score=+2
[L3-VT] [MSG_ID] attachment=invoice.pdf hash=abc123 malicious=0 score=+0
```

**Demo talking point:**
> "At the L3 stage, rather than a single blacklist lookup, PhishGuard submits every extracted URL and attachment hash to VirusTotal's aggregated intelligence network — the same feed used by enterprise SOC teams — and gets back a consensus verdict from over 70 scanning engines simultaneously. This directly mirrors Microsoft Defender's Safe Links detonation pipeline."

### Stage 6 — `pipeline/scoring.py`

```python
@dataclass
class PipelineResult:
    scl_score: int           # 0–10, capped
    verdict: str             # CLEAN / SUSPICIOUS / LIKELY_PHISH / PHISH
    l1_score: int
    l2_heuristics_score: int
    l2_nlp_score: int
    l3_score: int
    fired_rules: list[str]   # human-readable reasons from all stages
    nlp_label: str           # "phish" or "ham"
    nlp_confidence: float
    flagged_urls: list[str]
```

`aggregate_scores(...) -> PipelineResult` — sums all stage scores, caps at 10, assigns verdict tier.

---

### Stage 7 — `pipeline/orchestrator.py`

```python
async def run_pipeline(raw_mime: str, gmail_service, message_id: str) -> PipelineResult:
    # 1. Parse email → EmailContext
    # 2. L1 authentication check
    # 3. L2 heuristics check
    # 4. L2 NLP classification
    # 5. L3 VirusTotal MCP — three checks run concurrently:
    url_results, domain_result, file_results = await asyncio.gather(
        scan_all_urls(context.urls),
        scan_domain(context.sender_domain),
        scan_attachments(context.attachments),
    )
    # 6. Aggregate via scoring.py → PipelineResult
    # 7. Return result
```

- **Fail-safe:** catches all exceptions, returns `CLEAN` verdict with error log — never crashes the webhook
- Logs every stage: `[STAGE] [MSG_ID] score=X reason=Y`

## 11. Phase 2.5 — Warning Draft Injection *(planned, do not implement before Phase 2 is verified)*

### What it does
After `apply_verdict_labels()`, when verdict is `PHISH` or `LIKELY_PHISH`, inject a human-readable warning draft into the same Gmail thread so the user sees a full forensic breakdown without leaving their inbox.

### New file: `pipeline/warning_composer.py`

**Function:**
```python
async def inject_warning_draft(
    service,
    context: EmailContext,
    result: PipelineResult,
    user_id: str = "me"
) -> None
```

**Draft structure:**
1. Full-width red banner with ⚠️ PhishGuard name
2. SCL score as visual gauge: `█████████░ 9/10`
3. Bullet list of every rule from `PipelineResult.fired_rules`
4. NLP confidence score and label
5. Flagged URLs listed explicitly
6. Horizontal divider
7. Original email body rendered verbatim

**MIME construction:**
```python
msg["Subject"] = f"⚠️ [PHISHGUARD BLOCKED] — {context.subject}"
msg["In-Reply-To"] = context.message_id   # threads into original email
msg["References"] = context.message_id
raw = base64.urlsafe_b64encode(msg.as_bytes()).decode()
```

**Gmail API call:**
```python
service.users().drafts().create(
    userId=user_id,
    body={"message": {"raw": raw, "threadId": context.thread_id}}
).execute()
```
- Uses `users.drafts.create` — **does not send** the message
- No new Python dependencies — `email.mime` and `base64` are stdlib

**Integration point in `orchestrator.py`:**
```python
# After apply_verdict_labels():
if result.verdict in ("PHISH", "LIKELY_PHISH"):
    await inject_warning_draft(service=gmail_service, context=ctx, result=result)
```

**Pre-condition:** `EmailContext.thread_id` must be populated from `msg["threadId"]` in `main.py` when Phase 2 is built.

### Demo flow
```
1. Phishing email arrives in inbox
2. Pipeline fires — PHISH verdict, SCL 9/10
3. Email gets [PhishGuard] Phishing label → moved to Spam
4. Warning draft injected into same thread with red banner
5. Judge opens thread → sees original + full forensic breakdown
   (fired rules, NLP score, 72 engine VT verdict, flagged URLs)
```

## 13. Known Limitations

| Limitation | Impact | Fix |
|---|---|---|
| `last_history_id` is in-memory | App restart loses cursor; may miss/reprocess emails | Persist to Redis / SQLite |
| Watch expires every 7 days | Notifications stop | Cloud Scheduler renewal job |
| ngrok URL changes on restart | Pub/Sub subscription must be manually updated | Deploy to Cloud Run with stable URL |
| `token.json` is a local file | Breaks in stateless/containerised deployments | Store in Secret Manager |
| VT free tier: 4 req/min | Token bucket rate limiter handles this | Paid tier for production |
| NLP models must be pre-trained | `train_classifier.py` must be run once before startup | Include in setup docs |

## 14. Security Notes

- `credentials.json`, `token.json`, and `.env` are in `.gitignore` — **never commit them**
- The webhook at `/webhook/gmail` is unauthenticated in Phase 1/2 — Phase 3 should validate the `Authorization` header sent by Pub/Sub push (service account JWT)
- OAuth scope is `gmail.modify` — minimum needed. Do not escalate to `mail.google.com`
- `scan_file_hash` computes SHA-256 locally and queries VT by hash — **the file is never uploaded**
- VirusTotal API key stored in `.env` only — never hardcoded
